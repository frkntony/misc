Sharing Databases with Accounts

You can share productive SAP HANA and SAP ASE databases that are provisioned in an account with other accounts.

When you provision a database in an SAP Cloud Platform account, only this account has access to it. You can change this by using the cockpit and/or the console client and give other accounts controlled access to productive SAP HANA databases and SAP ASE databases that are owned by a different account. You can also allow other accounts to bind their Java applications to a database in a different account.

There are two different methods for sharing a database:
Method Description
Sharing Databases in the Same Global Account This method allows you to give an account permission to use a database that is owned by a different account. You can add and revoke this permission using the cockpit or the console client. See Managing Cross-Account Permissions.
Restriction

The account providing the permission and the account receiving the permission must be part of the same global account. For more information on global accounts, see Accounts.

You can set different types of permissions:
  • Permission for a data source binding
  • Permission to open a database tunnel
The account receiving the permission can bind its applications and/or open a tunnel to the database in the different account. See Binding Applications to Databases in the Same Global Account and Opening a Database Tunnel.
Sharing Databases with Other Accounts This method allows you to give any account permission to use a database that is owned by a different account. You can add and revoke this permission using the console client. See: Managing Access to Databases for Other Accounts
The account receiving the permission uses an access token to bind a Java application or to open a tunnel to a database in the other account. See:
  • Binding Applications to Databases in Other Accounts
  • Opening Tunnels to Databases in Other Accounts
