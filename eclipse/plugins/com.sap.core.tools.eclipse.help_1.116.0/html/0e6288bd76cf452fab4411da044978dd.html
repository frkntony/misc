Consuming an SAP HANA Database From a Virtual Machine

You can consume an SAP HANA database from a virtual machine using JDBC.

Prerequisites

  • You have created a virtual machine and connected to it over SSH protocol as a root user so that you can install your software. For more information, see Managing Virtual Machines.
  • Your account is configured with a dedicated SAP HANA database.

Procedure

  1. Install the runtime necessary to run your application on the virtual machine (for example, Java, Node.js).
  2. Get a valid JDBC driver for your application.
  3. To get the details required to connect to the database, go to the overview of the database in the SAP Cloud Platform cockpit.
    1. In the cockpit, select an account and choose Databases & Schemas.
    2. Select a database that you want to connect to. This opens the overview for the selected database.
  4. Create a database user to get access to an SAP HANA database. Please see Binding SAP HANA Databases to Java Applications (section: Create an SAP HANA Database User).
  5. To connect to the database, specify the following details:
    1. The user name and password that you defined earlier for the database.
    2. The host and port that you should take from the JDBC URL (for example, jdbc:sap://localhost:30015) displayed in the cockpit in the overview for the selected database.
