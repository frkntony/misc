Undeploying Solutions Using the Cockpit (Beta)

Prerequisites

Еnsure that the related HTML5 or Java applications is started, if your solution contains one or more of the following components:

  • SuccessFactors Connection
  • SuccessFactors Homepage Tile
  • SuccessFactors Application Access

Context

Note

This is a beta feature available for all SAP Cloud Platform accounts. For more information about the beta features, see Using Beta Features in Accounts.

To undeploy a solution from your account, proceed as described:

Procedure

  1. In the cockpit, choose Solutions (BETA) in the navigation area.
  2. In the solutuion list, select the tile of the solution you want to updeploy. The Delete a Solution dialog appears.
  3. Optionally, perform the following:
    1. Deselect the Delete data source bindings checkbox.
    2. Select the Clean-up on error checkbox. If set, any errors during deletion that are external to SAP Cloud Platform are ignored. External systems might still contain components that are not deleted.
  4. Select the Confirm solution deletion checkbox, and choose Delete.

    The solution undeployment dialog remains on the screen during the process. А confirmation appears when the undeployment is completed.

    If you close the dialog while the process is running, you can open it again by choosing Check Progress of the corresponding operation, located in the Ongoing Operations table in the solution overview page.

    Note
    • SFSF Roles are not deleted.
    • Custom application destinations and account destinations are also deleted.
    • Deleted solutions and their components cannot be restored.
