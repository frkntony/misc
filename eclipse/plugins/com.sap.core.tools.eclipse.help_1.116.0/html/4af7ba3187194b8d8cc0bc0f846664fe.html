upload-resource (Beta)

This command uploads a solution resource file into the SAP Cloud Platform system resource repository for a specified extension account.

Note

This is a beta feature available for SAP Cloud Platform extension accounts. For more information about the beta features, see Using Beta Features in Accounts.

neo upload-resource --name <resource_name> --location <path_to_resource> --account <account_name> --user <e-mail_or_user> --host <landscape_host> --overwrite
Parameters
To list all parameters available for this command, execute neo help upload-resource in the command line.
Required
-a, --account Account name

Type: string (up to 30 characters; lowercase letters and numbers, starting with a letter)
-h, --host Use the respective landscape host for your account type.

Type: URL. For acceptable values see Landscape Hosts.
-p, --password To protect your password, enter it only when prompted by the console client and not explicitly as a parameter in the properties file or the command line.

Type: string

-u, --user Use your email, SAP ID or user name

Type: string

-l, --location Path to the resource file in the local file system. Supported file formats are ZIP and GZIP.

Type: string
   
Optional
-n, --name Name of the resource to be uploaded

Type: string (must be unique on account level)

Default: file name of the resource
-w, --overwrite If a resource with the same name already exists, overwrites it. If you do not explicitly specify the --overwrite parameter, you will be notified and asked if you want to overwrite the resource.
Example
To upload a solution resource to the system repository for your account, execute:
neo upload-resource --name myresourcename --location localdrive:\Resources\roles.zip --account myextensionacc  --user mymail@example.com --host hana.ondemand.com
If a resource file with the same name roles.zip already exist in the repository, the system notifies you and asks you whether to overwrite it.
