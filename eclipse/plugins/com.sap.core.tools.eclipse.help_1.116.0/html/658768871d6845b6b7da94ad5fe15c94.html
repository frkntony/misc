Removing Password Prompt from Translation Workflow

Context

In your translation projects, users are requested to enter their password when performing the Get Translations activity. To remove the password prompt, follow the steps described below.

Procedure

  1. In the SAP Cloud Platform cockpit, choose Services in the navigation tree.
  2. Choose SAP Translation Hub from the list of services.
  3. In the service screen, choose Configure SAP Translation Hub (it is located under Service Configuration).
  4. In the screen that appears, choose New Destination and enter the following data:

    Field

    Entry
    Name gitConnectionConfig
    Type HTTP
    Description Enter a description of your choice, for example, Configuration for Git connection.
    URL

    Enter the Git URL for the HCP landscape you are in using the following pattern: https://git.<your HCP landscape>

    You can take the landscape from your HCP base URL. For example, if you have an account in HCP trial landscape (hanatrial.ondemand.com), you would enter https://git.hanatrial.ondemand.com.

    Proxy Type Internet
    Authentication BasicAuthentication
    User

    Enter the desired user in the following pattern: <valid HCP user name>,<e-mail ID for the same user>@<domain name>

    Example: username,username@sap.com

    Note This user must have a developer role in your account.
    Password Enter the password for the user above.
  5. Wait a bit. (It may take several minutes for the destination to become effective.)

Results

Next time you use the service, Get Translations will no longer prompt for a password.
Note With this configuration, all the operations from SAP Translation Hub to the Git server of your landscape (https://git.<your HCP landscape>.ondemand.com) are executed by the user you specify here, and not by the actual end user that is performing the action.
