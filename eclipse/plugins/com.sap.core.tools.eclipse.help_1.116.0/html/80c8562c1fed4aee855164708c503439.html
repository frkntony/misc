User for Connecting to the Document Service

An application has the following options to connect to the document service:

  • The application can use a connect method without explicitly providing a parameter containing a user. Then the current user is forwarded to the document service. The user's right to access particular documents and folders is determined using the user ID and the attached ACLs.
  • The application can provide a user ID explicitly using a parameter of the connect method. Then this ID is used for checking the access rights.
Note

Note that the document service is not connected to any Identity Provider or Identity Management System and considers the provided ID as an opaque string. This is also true for the user or principal strings provided in the ACEs when setting ACLs at objects.

The application is responsible for providing the correct user ID but it can also submit a technical user ID that does not belong to any physical user, for example, to implement some kind of service user concept.
