Roles

The following roles can be assigned to access the gamification service gamification workbench, API or demo app and must be explicitly assigned to a SAP Cloud Platform user:

Role Type Access Level Description

GamificationDesigner

User

Workbench

  • Summary

  • Game Design

  • Rule Engine

  • Terminal

  • Analytics

  • Full access to game mechanics and rules (read, write, activate etc.) for all apps created for the tenant

  • Partial access to the Rule Engine monitor, to read information about the health state of the current app's rule engine

  • Access to the AP Terminal, allowing to test the gamification concept directly via API

  • Read aggregated gamification analytics (no access to individual player data)

GamificationReviewer

 User

Workbench

  • Summary

  • Game Design (read-only)

  • Rule Engine

  • Analytics

  • Read game mechanics and rules (no write access)

  • Partial access to the Rule Engine monitor, to read information about the health state of the current app's rule engine

  • Read aggregated gamification analytics (no access to individual player data)

  • Export game mechanics and rules for a certain app

TenantOperator

User

Workbench

  • Game Design (read-only)

  • Rule Engine

  • Logging

  • Players

  • Operations

  • Managing Apps

  • Full access to the Rule Engine monitor with the ability to restore session snapshots

  • Read game mechanics and rules (no write access)

  • Access to the AP Terminal, allowing to test the gamification concept directly via API

  • Full access to user data, including player details with player achievements

  • Full access to team data

  • Full tenant export and import, including player data

AppStandard

Technical

API (methods are annotated with required role)

Terminal (send events for testing purposes)

  • Write only - using rules; reading achievements is possible, but should be avoided

  • Send player-related events

  • Read player achievements and available achievements

Sample use cases:

  • Show aggregated gamification statistics in a host application

  • Visualize specific leaderboards in a host application

AppAdmin

Technical

API (methods are annotated with required role)

  • Read and delete a player record for a single app or for the whole tenant

  • Create and delete a user or a team

Player (automatically assigned)

Technical (implicit role)

API (methods are annotated with required role)

  • Send player-related events (only works for the user that is authenticated using the identity provider which is configured for your account)

Note

This role is not a standard SAP Cloud Platform role. It is automatically assigned to a user (player) that is created using the gamification service and cannot be explicitly assigned to a SAP Cloud Platform user.

helpdesk

User

Demo App

  • Access to demo app HelpDesk
