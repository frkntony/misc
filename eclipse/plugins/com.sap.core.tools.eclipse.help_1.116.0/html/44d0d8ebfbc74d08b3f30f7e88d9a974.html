Changing Cross-Account Permission Types

You use the cockpit to change the type of an existing cross-account permission.

Prerequisites

  • You have the administrator role in the account that owns the database.
  • You have given an account permission to use a database that is owned by another account. The account providing the permission and the account receiving the permission are part of the same global account. See Adding New Cross-Account Permissions.

Procedure

  1. Log on to the cockpit with the administrator role and select the account that owns the database for which you would like to change permissions.
  2. Choose  Persistence  Databases & Schemas  in the navigation area.
  3. Choose the required database.
  4. Choose the Edit permission icon for an existing permission.
  5. Change the available permission options as required. Select the TUNNEL checkbox for example and deselect the BINDING checkbox to change the permission type from Binding to Tunnel.
  6. Choose Save.

Results

You have changed the type of a cross-account permission.
