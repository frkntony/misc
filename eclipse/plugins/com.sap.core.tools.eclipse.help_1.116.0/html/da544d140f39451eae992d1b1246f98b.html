Translation Workflow with SAP Translation Hub

SAP Translation Hub supports the translation of short texts in different scenarios.

You can use SAP Translation Hub to translate texts from the following sources:

Source of Texts to Be Translated Details
HTML5 applications from the Git repository of SAP Cloud Platform

The integration of SAP Translation Hub with Git projects enables you to translate texts in source properties files for HTML5 applications.

You can use this workflow if you want to translate the texts in HTML5 applications that are stored in the Git repository of SAP Cloud Platform.

Each HTML5 application comprises at least one properties file that contains texts that require translation. The translation workflow reads the texts in a source properties file from the Git repository, translates the texts using one of the translation providers in SAP Translation Hub, and stores the translations in language-specific properties files in the Git repository.
ABAP-based repositories using a remote function call (RFC)

The integration of SAP Translation Hub with ABAP systems enables you to translate short texts from ABAP objects such as data elements, message classes, and reports.

You can use this workflow to leverage SAP Translation Hub’s translation providers, such as the multilingual text repository and machine translation, in your ABAP-based applications.

By using an RFC connection, you can import source texts from object lists in ABAP systems, and then push the translations back to your ABAP system.
Manual file upload
The following file formats are supported:
  • Java properties
  • JSON
  • ABAP-based XLIFF files, such as the files that you can generate with the export features of SAP NetWeaver

You can use this workflow to leverage SAP Translation Hub’s translation providers, before you import the translated files in to your original application.
Translation Projects

To manage the translation workflow, SAP Translation Hub requires some information about your application. For example, for an HTML5 application, it needs the location of the source properties file in the Git repository and the required target languages. For ABAP-based translations, a remote function call (RFC) must be set up. For information about how to create a translation project, see the Related Information section.

Main Steps of Translation Workflow
When you start the translation workflow, SAP Translation Hub does the following:
  1. Reads the translation project.
  2. Gets translations from the following sequence of translation providers:
    1. Multilingual text repository in SAP Translation Hub
    2. SAP-internal machine translation solution based on the Moses open source toolkit (SAP machine translation)
  3. Enters translations in the source repository or in an export file.
