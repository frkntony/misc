edit-hanaxs-destination

This command (beta) sets or clears a trust store for a specified destination.

neo edit-hanaxs-destination --account <account_name>  --host <landscape_host> --user <e-mail_or_SCN_user> --package <package> --destination <destination_file_path> --truststore default
Restriction The command only refers to trial accounts, thus you can only execute in on the trial landscape.
Parameters

To list all parameters available for this command, execute neo help edit-hanaxs-destination in the command line.

Required
-a, --account Account name

Type: string (up to 30 characters; lowercase letters and numbers, starting with a letter)
--destination HANA XS destination. It represents the relative path to the destination file within the package.
-h, --host You can only use it on hanatrial.ondemand.com.
--package HANA root package
-p, --password To protect your password, enter it only when prompted by the console client and not explicitly as a parameter in the properties file or the command line.

Type: string

--truststore Name of the trust store

Acceptable values: default

-u, --user Your e-mail, SAP ID or SCN user name

Type: string
Example
neo edit-hanaxs-destination --account myaccount --host hanatrial.ondemand.com --user mymail@example.com --package mypackage --destination mysubpackage/mydest --truststore default
