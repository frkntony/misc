Updating and Exporting Translations in Individual Files

You can use the SAP Translation Hub UI to add or change translations created by SAP Translation Hub during the translation workflow.

Prerequisites

You have used the translation workflow of SAP Translation Hub to translate the texts from a file that you have uploaded manually.

Context

When you use the translation workflow, SAP Translation Hub stores the translations of texts from your source file for each target language. If required, you can create or update translations, and finally export them to a file. When exporting your translations, you get one file per target language (or per language pair for XLIFF files).

Procedure

  1. On the SAP Translation Hub UI (<base URL of SAP Translation Hub>/ui), choose the translation project that refers to the uploaded source file whose translations you want to update and export.
  2. Choose the Translations tab.

    SAP Translation Hub shows the languages defined in the translation project along with the source texts from the object list and the translations.

  3. Choose the target language in which you would like to check or update translations.
  4. In the Translated Text column, update translations as required. If SAP Translation Hub didn't find a translation for some texts, enter your own translation.

    The translation provider is updated to show that you changed the text that SAP Translation Hub entered.

  5. Save your changes.

    If you need to change the translations in other target languages, repeat the previous steps for each target language.

  6. To export your translations to the corresponding file format, choose Download Translations.

Results

A .zip file is generated containing one file of the corresponding format per target language (or per language pair for XLIFF files). You can then download the .zip file and then manually upload the target files into your original application.
