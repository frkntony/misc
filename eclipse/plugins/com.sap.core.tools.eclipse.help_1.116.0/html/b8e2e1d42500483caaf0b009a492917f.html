Developing with SAP HANA Web-based Development Workbench

Open the SAP HANA Web-based Development Workbench from the cockpit.

Context

The SAP HANA Web-based Workbench includes an all-purpose editor tool that enables you to maintain and run design-time objects in the SAP HANA repository.

Caution Make sure that you do not grant access to your artifacts to PUBLIC role. All items granted to PUBLIC role will be deleted.

Procedure

  1. Log on to the cockpit on the trial landscape and select an account.
  2. Choose Database Systems in the navigation area.

    All database systems available in the account are listed with their details, including the database type, version, memory size, state, and the number of associated databases.

  3. To select the entry for the relevant database system in the list, click the link on its name.
    The overview of the database system shows details, including the database version and state, and the number of associated databases.
  4. Choose Databases in the navigation area.
  5. To select the relevant database in the list, click the link on its name.
  6. In the overview of the selected database, click SAP HANA Web-based Development Workbench under Development Tools.
    You will be logged on with an automatically created SAP HANA developer user (prefixed DEV_). This user will be shown in the title bar of the workbench screen.
    Note If you are inactive for some time, your session in the SAP HANA Web-based Development Workbench will become invalidated. To start a new session, go back to the cockpit and make sure that your session is active there. Open the SAP HANA Web-based Development Workbench from there as described in the current topic.
