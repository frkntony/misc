Creating Translation Projects for Individual Files

SAP Translation Hub enables you to upload and translate files in various formats.

Prerequisites

  • The source language of the file is English.
  • The encoding of the file is UTF-8.

Context

You can import files in the following formats:
  • iOS strings files
  • Java properties files
  • JSON files with the following structure:
    Code Syntax 
    {
	"key1" : "value1"
	," key2" : " value2"
}
    For example:
    Sample Code 
    {
	"okButton" : "OK"
	,"cancelButton" : "Cancel"
}
  • ABAP-based XLIFF files that are generated using the export features of SAP NetWeaver. For information about exporting and importing XLIFF files in ABAP systems, see the following topic on SAP Help Portal at http://help.sap.com/nw under  SAP NetWeaver Platform  <your NetWeaver version>  Application Help  Function-Oriented View . Then follow the path listed below based on your SAP NetWeaver version:
    • 7.3 - 7.5:

       Application Server  Application Server ABAP  Other Services  Services for Information Developers and Translators  Setting Up and Coordinating Translation (BC-DOC-TTL)   

    • 7.0 including Enhancement Package 3:

       Application Platform by Key Capability  Application Server ABAP  Other Services  Services for Information Developers and Translators  Setting Up and Coordinating Translation (BC-DOC-TTL) 

    • 7.0 - 7.0 including Enhancement Package 2:

       Application Platform by Key Capability   ABAP Technology  Documentation and Translation Tools (BC-DOC)  Setting Up and Coordinating Translation (BC-DOC-TTL) 

    Note
    More specifically, check the details in the following topics:
    • Exporting Texts for Offline Translation
    • Importing Texts After Offline Translation

Procedure

  1. Create a translation project by choosing File Upload as the type.
  2. Enter the following data:

    Field

    Entry
    Project Name Enter your project name.
    File Type Select the file type from the list of supported formats.
    Domain Select the translation domain that most closely matches the application area of your source texts, for example, Controlling or Personnel Management.
    Target Languages Choose the target languages into which you want to translate the texts in your source file.
  3. Save the translation project.
  4. Upload your file by selecting it using the value help or by dragging and dropping the file directly onto the Details screen of your project.

Results

Your translation project contains the data required to translate the texts in your source file.
Note
Once you have created your project, note the following:
  • You cannot change the source file type.
  • You can add more target languages, but to enable SAP Translation Hub to register additional target languages, you must upload the source file again.
  • You can upload the source file as many times as you want, and even replace the source file with a different file. However, if you upload a different file, your project’s source texts will be replaced with the source texts from the new file that you upload. Any translations that you have done in the project will be deleted. To avoid losing your work, export your translations before you upload a different source file. For more details, see Adding Translations and Updating and Exporting Translations in the Related Information section.
  • If you use ABAP-based XLIFF files, there are usually multiple .xlf files, because ABAP systems generate one file per language pair during the export process. Upload only one of the files; it does not matter which file you upload. SAP Translation Hub does not import any of the translations that the XLIFF files contain.
