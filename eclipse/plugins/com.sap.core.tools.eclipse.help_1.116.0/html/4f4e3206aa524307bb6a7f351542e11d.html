Translating ABAP Short Texts Using an RFC

You use translation projects to translate short texts created in ABAP systems as part of the translation workflow of SAP Translation Hub.

Prerequisites

There is a translation project for the ABAP object list that contains the texts to be translated.

Context

On the SAP Translation Hub UI (<base URL of SAP Translation Hub>/ui), choose the translation project that refers to the object list to be translated, and choose one of the translation options in the bottom right of the screen:
What Do You Want to Do? Choose this Option Follow-On Tasks
Translate the texts and update the ABAP system straightaway; I don't need to review them before updating the ABAP system.
Note Any existing translations of your texts in the ABAP system will be overwritten.
 Get Translations You can nevertheless check the translations on the Translations tab.
Translate the texts and review them. Some texts are tricky, so I want to make sure the translations are up to scratch before updating the ABAP  Get Translations  Translate Texts for You to Review  Review the translations on the Translations tab. After you're satisfied with the translations, push them to the ABAP system by choosing  Get Translations  Push Translations to Repository .

For an overview of how to translate ABAP short texts, check out the following video on YouTube: SAP Translation Hub - Translating Texts from ABAP Systems Using an RFC

Results

The source texts are translated into the target languages specified in the translation project.

If you want to view or change the translations in the project, switch to the Translations tab. You see all the translations in the translation project and the translation provider SAP Translation Hub uses for each text.
You also see the quality index, which helps you to judge the quality of translations. The higher the number on a scale from 0-100, the better the quality. The quality index is based on the following:
  • Ratings that language specialists assign to translations
  • The number of instances in the multilingual text repository in which the combination of a source text and a particular translation occurs. The more often a given combination of source text and translation occurs, the more confident we are that the translation is good.
Note The quality index currently appears for translations that are provided by the multilingual text repository or your ABAP system only.

If you change any of the translations, you must save your changes and push the changes to the ABAP system manually.

Note An ABAP system object can contain more than one text element. For example, a data element contains multiple labels of different lengths and a short description. In your translation projects, you see all text elements that belong to a single object, and all text elements have the same key. The key is the object name in the ABAP system. SAP Translation Hub does not provide any context details for the different text elements in the objects, such as character length or label type.
